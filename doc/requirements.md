# Video Mosaic Tool - 要件定義書

**プロジェクト名**: Video Mosaic Tool  
**目的**: バスケットボール観戦動画から観客の顔をモザイク化し、プライバシーを保護する  
**ユーザー**: obiさん（個人用）  
**優先度**: 高  

---

## 1. 機能要件

### 1.1 主要機能

#### F1: 動画ファイルの選択・アップロード
- **説明**: ユーザーがローカルファイルから動画（MP4、WebMなど）を選択
- **入力**: MP4, WebM, MOV など一般的な動画形式
- **出力**: 選択された動画ファイル → プレビュー表示
- **制約**: ファイルサイズは 100MB 以下推奨（WASM処理のため）
- **処理**: 完全ローカル処理（サーバーへのアップロードなし）

#### F2: 動画プレビュー表示
- **説明**: 選択した動画をHTMLVideoタグで再生可能にする
- **機能**:
  - 再生・一時停止・スクラバー操作
  - 動画の解像度・長さを自動認識
  - 動画上に透明Canvas オーバーレイ（領域選択用）

#### F3: ぼかし領域の選択（UI）
- **説明**: ユーザーが動画プレビュー上でドラッグして四角形を描画
- **操作**:
  - マウス: クリック＆ドラッグ
  - タッチ: 指でドラッグ（スマホ対応）
- **視覚フィードバック**:
  - ドラッグ中: **青い枠**（プレビュー）
  - 確定後: **黄色い枠**（確定領域）
- **複数領域**: 1つの領域のみサポート（複雑性回避）
- **領域変更**: 領域選択をやり直せる

#### F4: ぼかし強度の調整
- **説明**: スライダーでぼかし強度を 0～100% で指定
- **表示**: 
  - 現在値（%）表示
  - 対応する blur kernel サイズ表示（例：50% = kernel 25）
- **デフォルト値**: 50%
- **リアルタイム更新**: スライダー動作時に即座に表示更新

#### F5: モザイク処理実行
- **説明**: FFmpeg で動画全体をぼかし処理
- **処理内容**:
  - FFmpeg WASM で動画をデコード
  - `boxblur=X:X` フィルタを適用（全体ぼかし）
  - H.264 でエンコード
  - MP4 として出力
- **処理時間**: 初回 30～60秒、2回目以降キャッシュで高速化
- **プログレス表示**: 0～100% のプログレスバー
- **エラーハンドリング**: 処理失敗時はアラート表示

#### F6: 動画ダウンロード
- **説明**: 処理済み動画を自動ダウンロード
- **ファイル名**: `blurred_${originalFileName}`
- **形式**: MP4（H.264 video + AAC audio）
- **自動トリガー**: 処理完了時に自動ダウンロード開始

## 2. 非機能要件

### 2.1 性能
- **初期化時間**: FFmpeg WASM ロード 10～30秒（1回限り）
- **処理時間**: 
  - 1分動画: 20～40秒
  - 10分動画: 3～5分
- **メモリ使用**: 500MB～2GB（動画サイズに依存）

### 2.2 互換性
- **ブラウザ**: 
  - Chrome / Edge / Firefox（最新版）
  - Safari 14+
  - スマートフォン: iOS Safari, Chrome Mobile
- **WASM対応**: SharedArrayBuffer 必須（Vercel で利用可能）

### 2.3 セキュリティ
- **プライバシー**: 完全ローカル処理（サーバーアップロードなし）
- **データ削除**: 処理後、メモリ内のファイルは即座に削除（FFmpeg.deleteFile）
- **HTTPS**: Vercel 自動HTTPS

### 2.4 UX
- **言語**: 日本語
- **デザイン**: 
  - グラデーション背景（青系）
  - 丸みを帯びたボタン・カード（Tailwind CSS）
  - レスポンシブデザイン（モバイル対応）
- **インタラクション**:
  - ボタンは disabled / enabled を視認
  - ローディング中は「⏳ 処理中...」表示
  - エラーは日本語メッセージ

## 4. UI / UXフロー

### 4.1 画面遷移

```
[スプラッシュ画面]
  ↓ (FFmpeg 初期化中, ローディングバー表示)
  ↓ (100% → 自動遷移)
[メイン画面]
  ├─ セクション1: 動画選択
  │   └─ ファイルインプット
  ├─ セクション2: プレビュー＆領域選択（動画選択後表示）
  │   ├─ 動画プレビュー（再生可能）
  │   └─ Canvas オーバーレイ（ドラッグで領域選択）
  ├─ セクション3: ぼかし強度調整（動画選択後表示）
  │   └─ スライダー
  └─ セクション4: 処理実行（領域選択後有効）
      └─ 「ぼかし処理 & ダウンロード」ボタン
         ↓ (クリック)
      [処理中画面]
         └─ プログレスバー（0～100%）
         ↓ (100% → 自動ダウンロード)
      [ダウンロード完了]
```

### 4.2 領域選択UI詳細

**状態1: ドラッグ中**
- Canvas に青い枠が描画
- 枠は動画上に正確に重ねられている

**状態2: 確定後**
- 黄色い枠に変わる
- テキスト「✅ 領域を選択しました」表示
- 「ぼかし処理」ボタン有効化

**状態3: 再選択**
- 黄色い枠をクリック or ドラッグで新規選択開始
- 前の選択は上書き

## 5. エラーハンドリング

| エラーケース | メッセージ | 対応 |
|-------------|----------|------|
| FFmpeg 初期化失敗 | 「FFmpeg の初期化に失敗しました」 | メイン画面を表示（スキップ可能） |
| 動画ファイル選択なし | 「動画を選択してください」 | ボタン disabled |
| 領域選択なし | 「ぼかし領域を指定してください」 | ボタン disabled |
| FFmpeg 処理失敗 | 「処理中にエラーが発生しました。ブラウザコンソールを確認してください」 | アラート表示 |
| ブラウザ非対応 | 「このブラウザは対応していません（SharedArrayBuffer が必要）」 | 注意メッセージ |

## 7. 今後の拡張案（v2以降）

### 7.1 複数領域対応
- ユーザーが複数の領域を選択可能
- 時間帯ごとに異なる領域をぼかし
- 現在の実装では非常に複雑（FFmpeg filter_complex）

### 7.2 顔自動検出
- face-api.js などで自動顔検出
- ぼかし領域を手動選択不要
- 処理時間がさらに長くなる

### 7.3 部分ぼかし（crop+blur+overlay）
- 指定領域のみぼかし、背景は保持
- 現在の「全体ぼかし」より高度
- FFmpeg filter_complex が複雑
